# Apple911 AI 開發規範

## 專案性質
本專案是 Apple911 既有網站系統修正與功能擴充。

本專案包含：

### 前台系統
- 首頁
- 商品頁
- 購物車
- 結帳流程
- 訂單流程

### 後台系統
- 商品管理
- 訂單管理
- 工單管理
- 報表系統
- 庫存功能
- 利潤與成本計算

技術架構：
- HTML
- Tailwind CSS
- JavaScript
- localStorage

---

## 回覆規則
- 優先使用繁體中文回答
- 回覆直接，不要過度冗長
- 不要重複解釋已經確認過的事情
- 如果是 debug，先分析問題來源再修改

## 開發規則
- 修改程式時優先使用最小修改
- 優先輸出 diff 而不是完整檔案
- 不要擅自重構專案
- 不要擅自修改 UI
- 不可修改 Tailwind 設定
- 不可更動 class 名稱
- 不可重新設計架構
- 不可新增未要求功能
- 不可刪除既有功能

---

## 修改前必須
- 說明會修改哪些 function
- 說明修改原因
- 說明可能影響範圍

## UI 規則
禁止：
- 修改顏色
- 修改排版
- 修改字體
- 修改 icon
- 修改按鈕樣式
- 修改動畫
- 修改原有操作邏輯

任何未經要求的 UI 變動都視為錯誤。

---

## GitHub 工作流
- main 為正式站
- dev 為 AI 修改測試站
- AI 應優先修改 dev branch
- 測試後再 merge 到 main

---

## Git / Deploy Rules

- 每次完成修改後，直接建立 PR 並 merge 到 main
- 不要停留在 claude/* 分支
- main 是正式部署分支
- Vercel 會讀取 main 自動部署
- 修改完成後不需要再詢問我是否 merge
- merge 前必須確認登入、商品管理、訂單、購物車、結帳流程沒有被破壞
- 不得重構專案
- 不得修改 UI
- 只針對我指定的問題做最小修改
